# Parc Auto — Guide Utilisateur Complet

## Introduction

Parc Auto est une application web de gestion de flotte de véhicules de location. Elle permet à l'administrateur de gérer l'intégralité du cycle de vie des véhicules : de l'enregistrement à la location, en passant par la maintenance, les assurances, le contrôle technique, le suivi de carburant, et le suivi du kilométrage (compteur km).

L'application est composée d'un backend ASP.NET Core (API REST) avec une base de données SQLite, et d'un frontend Angular 19 avec des composants PrimeNG. L'interface est bilingue (français et anglais) avec un sélecteur de langue intégré dans la barre de navigation.

### Architecture Technique
- **Backend** : ASP.NET Core Web API (.NET 8), SQLite, Entity Framework Core
- **Frontend** : Angular 19 (standalone components), PrimeNG, Chart.js
- **Authentification** : JWT (JSON Web Token)
- **Port Backend** : http://localhost:5222
- **Port Frontend** : http://localhost:4200

### Démarrage de l'application
Pour lancer l'application, exécutez le fichier `start.bat` à la racine du projet. Ce script démarre simultanément le serveur backend et le serveur frontend de développement.

## Module 0 — Connexion (Login)

### Page de Connexion
L'application nécessite une authentification. L'écran de connexion est la première page affichée.

**Identifiants par défaut :**
- **Nom d'utilisateur** : `admin`
- **Mot de passe** : `AdminPassword123!`

### Fonctionnement
1. Saisissez votre nom d'utilisateur dans le champ "Nom d'utilisateur"
2. Saisissez votre mot de passe dans le champ "Mot de passe"
3. Cliquez sur **Connexion**
4. En cas de succès, vous êtes redirigé vers le **Tableau de bord**
5. En cas d'erreur, un message s'affiche sous le formulaire

Le token JWT est stocké dans le localStorage du navigateur. Toutes les requêtes API ultérieures incluent ce token dans l'en-tête `Authorization: Bearer <token>`.

### Déconnexion
- Cliquez sur le bouton **Déconnexion** (icône de sortie) dans la barre latérale de navigation
- Le token est supprimé et vous êtes redirigé vers la page de connexion

## Module 1 — Tableau de Bord (Dashboard)

### Accès
- Route : `/dashboard`
- Menu de navigation : première entrée dans la barre latérale (icône de tableau de bord)

### Vue d'ensemble
Le tableau de bord offre une vue synthétique de l'état de la flotte et de l'activité en cours. Il affiche :

#### Indicateurs clés (KPI Cards)
1. **Flotte** : Nombre total de véhicules enregistrés dans le système
2. **Locations actives** : Nombre de véhicules actuellement en location, avec le nombre de véhicules disponibles en sous-indicateur
3. **Chiffre d'affaires** : Revenu total généré (montant total), avec le montant encaissé en sous-indicateur
4. **Alertes** : Nombre total d'alertes actives, avec le nombre d'alertes critiques en sous-indicateur rouge

#### Graphique de disponibilité de la flotte
- Graphique en anneau (doughnut) montrant la répartition des véhicules par statut :
  - **Disponible** (vert)
  - **Loué** (bleu/accent)
  - **En maintenance** (orange)
  - **Réservé** (bleu clair)
  - **Immobilisé** (rouge)

#### Alertes prioritaires
- Liste des alertes les plus récentes et les plus critiques
- Chaque alerte affiche : la cible concernée (véhicule ou client), le message d'alerte, et un badge de sévérité (Critique / Avertissement / Info)
- Lien **Voir tout** pour accéder au Centre d'alertes complet

#### Contrats impayés
- Tableau listant les contrats dont le paiement est en attente (statut Impayé ou Partiellement payé)
- Colonnes : N° de contrat, Client, Véhicule, Date de départ, Montant, Statut de paiement
- Lien **Gérer** pour accéder directement au contrat concerné dans le module Contrats

## Module 2 — Gestion des Véhicules

### Accès
- Route : `/vehicles`
- Menu de navigation : **Véhicules** (icône de voiture)

### Liste des véhicules
Les véhicules sont affichés sous forme de **grille de cartes** (cards), chacune montrant :
- Photo du véhicule (ou un placeholder si aucune photo n'est uploadée)
- Badge de statut superposé sur la photo (Disponible, Loué, En maintenance, Réservé, Immobilisé)
- Tarif journalier affiché sur la photo
- Marque et modèle + badge de l'année
- Numéro de matricule (plaque d'immatriculation)
- Spécifications : carburant, transmission, kilométrage actuel

### Filtres et recherche
La barre de filtres en haut de la page permet de :
- **Rechercher** par texte libre (matricule, marque, modèle)
- **Filtrer par type** : Voiture, SUV, Van, Camion, Moto, etc.
- **Filtrer par carburant** : Essence, Diesel, Électrique, Hybride, GPL
- **Filtrer par statut** : Disponible, Loué, En maintenance, Réservé, Immobilisé

### Pagination
Si le nombre de véhicules dépasse la limite par page, des contrôles de pagination sont affichés en bas.

### Ajouter un véhicule
1. Cliquez sur le bouton **+ Ajouter un véhicule** en haut à droite
2. Un formulaire modal s'ouvre avec deux colonnes :
   - **Colonne gauche** : Marque, Modèle, Matricule, VIN (numéro de châssis), Année, Couleur, Type de véhicule, Nombre de places, Type de carburant, Transmission
   - **Colonne droite** (fond gris clair) : Tarif journalier, Prix d'achat, Numéro moteur, Kilométrage initial, Photo du véhicule (zone de dépôt cliquable avec aperçu)
   - **Pleine largeur** : Remarques / notes
3. Cliquez sur **Enregistrer** pour créer le véhicule

**Upload de photo :**
- Cliquez dans la zone de dépôt ou faites glisser une image
- Formats acceptés : PNG, JPG (max 5 Mo)
- Un aperçu de la photo s'affiche après upload
- En mode édition, survolez la photo pour afficher le bouton de remplacement

### Modifier un véhicule
1. Sur la carte du véhicule, cliquez sur l'icône **crayon** (éditer)
2. Le même formulaire modal s'ouvre, pré-rempli avec les données actuelles
3. Le champ **Kilométrage initial** est désactivé en mode édition (il ne peut pas être modifié après la création)
4. Modifiez les champs souhaités et cliquez sur **Enregistrer**

### Supprimer un véhicule
1. Sur la carte du véhicule, cliquez sur l'icône **poubelle** (supprimer)
2. Une confirmation est demandée
3. **Important** : Un véhicule ne peut pas être supprimé s'il a des contrats actifs ou en brouillon

### Dossier du véhicule (Vehicle Folder / Tracking)
Chaque véhicule possède un dossier complet accessible en cliquant sur le bouton **Dossier** sur sa carte.

Un modal détaillé s'ouvre avec **5 onglets** :

#### Onglet 1 : Consommables
Affiche l'état de tous les consommables suivis pour ce véhicule :
- **Colonnes** : Type de consommable, Dernier remplacement (date et km), Distance parcourue / mois écoulés, Intervalle de remplacement, Statut
- **Statuts** : OK (vert), Avertissement (orange), Dû (rouge)
- Les types de consommables incluent : Vidange huile, Filtre à huile, Filtre à air, Filtre à carburant, Filtre habitacle, Freins avant/arrière, Pneus avant/arrière, Batterie

#### Onglet 2 : Assurances
- Historique de toutes les polices d'assurance du véhicule
- **Colonnes** : Assureur, N° de police, Type de couverture, Validité (dates), Prime / Valeur assurée, Statut
- **Statuts** : Valide (vert), Bientôt expiré (orange), Expiré (rouge)
- Bouton **+ Ajouter** pour créer une nouvelle police
- Formulaire d'ajout : Nom de l'assureur, N° de police, Type de couverture (Tiers, Tous risques, Flotte), Prime, Dates d'effet et d'expiration, Document PDF

#### Onglet 3 : Contrôle Technique
- Historique des inspections techniques du véhicule
- **Colonnes** : Date, Expiration, Centre de contrôle, Résultat, Coût, Statut
- **Résultats** : Favorable (vert), Contre-visite, Défavorable (rouge)
- **Statuts** : Valide, Bientôt expiré, Expiré, Échoué
- Bouton **+ Ajouter** pour enregistrer une nouvelle inspection
- Formulaire d'ajout : Date d'inspection, Date d'expiration, Résultat, Coût, Centre et adresse, Remarques, Document PDF

#### Onglet 4 : Carburant
- Journal de tous les pleins de carburant du véhicule
- **Colonnes** : Date, Compteur km, Volume (litres + prix/L), Coût total, Consommation (L/100km calculée automatiquement), Anomalie
- **Anomalie** : Un badge rouge "Surconsommation" s'affiche si la consommation dévie significativement de la moyenne du véhicule
- Bouton **+ Plein** pour ajouter un nouveau plein de carburant
- Bouton **CSV** pour exporter le journal carburant en fichier CSV
- Formulaire d'ajout : Date, Compteur km, Volume en litres, Prix par litre, Station

#### Onglet 5 : Historique Kilométrage
- Chronologie (timeline) de toutes les entrées kilométriques du véhicule
- Chaque entrée affiche : date/heure, valeur du compteur km, notes, source
- **Sources** : Manuel (point vert), Départ contrat (point bleu accent), Retour contrat (point bleu info)
- Bouton **+ Relevé** pour ajouter une entrée manuelle de kilométrage
- Bouton **CSV** pour exporter l'historique en fichier CSV
- Formulaire d'ajout : Date, Compteur km, Notes

## Module 3 — Gestion des Clients

### Accès
- Route : `/clients`
- Menu de navigation : **Clients** (icône de personnes)

### Liste des clients
Les clients sont affichés dans un **tableau** avec les colonnes :
- Nom complet
- N° d'identité nationale / Passeport
- Téléphone, Email
- N° de permis + catégorie
- Expiration du permis
- Actions (éditer, supprimer)

### Recherche
- Barre de recherche pour filtrer par nom, identifiant national, ou numéro de permis

### Ajouter un client
1. Cliquez sur **+ Nouveau client**
2. Remplissez le formulaire :
   - Nom complet (requis)
   - N° d'identité nationale ou passeport (requis, unique)
   - Date de naissance
   - N° de permis de conduire, catégorie (ex: B), date de délivrance, date d'expiration
   - Téléphone(s), email, adresse
   - Notes / remarques
3. Cliquez sur **Enregistrer**

### Alerte de permis expiré
Lorsqu'un contrat est créé ou modifié, le système vérifie automatiquement la date d'expiration du permis du client. Si le permis est expiré, un **bandeau d'avertissement** jaune s'affiche en haut du formulaire de contrat pour prévenir l'administrateur.

### Historique de location
L'historique des locations passées et en cours d'un client est visible via le module Contrats (en filtrant par le nom du client).

## Module 4 — Contrats de Location

### Accès
- Route : `/contracts`
- Menu de navigation : **Contrats** (icône de documents)

### Liste des contrats
Les contrats sont affichés dans un **tableau** avec les colonnes :
- N° de contrat (auto-généré, format : CTR-YYYYMMDD-NNN)
- Client
- Véhicule (marque, modèle + matricule)
- Période (date de début → date de retour)
- Montant net dû
- Double badge de statut : statut du contrat + statut du paiement
- Actions

### Statuts de contrat
- **Brouillon** (gris/info) : contrat créé mais pas encore activé
- **Actif** (orange/warning) : location en cours
- **Terminé** (vert/success) : véhicule retourné
- **Annulé** (rouge/danger) : contrat annulé

### Statuts de paiement
- **Impayé** (rouge) : aucun paiement reçu
- **Partiellement payé** (orange) : paiement partiel reçu
- **Payé** (vert) : paiement complet reçu

### Filtres
- **Recherche** par texte libre (N° de contrat, nom du client, matricule)
- **Filtre par statut** : Tous, Brouillon, Actif, Terminé, Annulé
- **Filtre par paiement** : Tous, Impayé, Partiellement payé, Payé

### Créer un nouveau contrat
1. Cliquez sur **+ Nouveau contrat**
2. Le formulaire modal s'ouvre avec deux colonnes :

**Colonne gauche :**
- **Client** : Sélectionnez dans la liste déroulante (affiche nom + identifiant)
- **Véhicule** : Sélectionnez parmi les véhicules disponibles/réservés (affiche marque, modèle, matricule)
- **Date et heure de départ** : Date picker avec sélecteur d'heure
- **Date et heure de retour prévue** : Date picker avec sélecteur d'heure
- **Type de contrat** : Journalier, Hebdomadaire, Mensuel, Long terme
- **Tarif journalier** : Pré-rempli depuis la fiche véhicule, modifiable
- **Statut initial** : Brouillon (réservation) ou Actif (activer immédiatement)
- **Facturation initiale** : Impayé, Acompte versé, Payé intégralement
- **Notes administrateur**

**Colonne droite (fond gris) :**
- **Jours calculés** : Nombre de jours automatiquement calculé entre les dates
- **Montant location** : Tarif journalier × nombre de jours
- **Frais supplémentaires** : GPS, siège enfant, etc.
- **Remise** : Montant de la réduction
- **Mode de paiement** : Espèces, Carte, Virement, Chèque
- **Montant payé** et **Caution (dépôt)**

**Pied du formulaire :**
- **Montant net total dû** : Calculé automatiquement
- **Reste dû** : Affiché en rouge si paiement partiel

3. Cliquez sur **Enregistrer le contrat**

### Comportement automatique à l'activation
Lorsqu'un contrat passe au statut **Actif** :
- Le statut du véhicule passe automatiquement à **Loué**
- Le kilométrage de départ est enregistré automatiquement dans le suivi kilométrique
- Une entrée kilométrique est créée avec la source "Départ contrat"

### Retourner un véhicule (clôturer un contrat)
1. Sur un contrat **Actif**, cliquez sur le bouton vert **Retour** (icône de clé)
2. Un formulaire modal s'ouvre affichant :
   - Kilométrage de départ du contrat (lecture seule)
   - Date de retour prévue (lecture seule)
3. Remplissez :
   - **Compteur kilométrique au retour** (requis)
   - **Date et heure de retour réelle**
   - **Pénalité carburant** (montant si le réservoir n'est pas plein)
   - **Frais de dommages** (si le véhicule présente des dégâts)
   - **Case à cocher : Envoyer en maintenance** (si le véhicule nécessite une intervention)
   - **Notes de retour**
4. Cliquez sur **Valider le retour**

### Comportement automatique au retour
- Le contrat passe au statut **Terminé**
- La date de retour réelle et le km au retour sont enregistrés
- Les **frais de retard** sont calculés automatiquement si applicable (retour après la date prévue)
- Le **kilométrage parcouru** (km retour − km départ) est calculé
- Une entrée kilométrique est créée avec la source "Retour contrat"
- Le statut du véhicule revient à **Disponible** (ou **En maintenance** si la case correspondante est cochée)

### Impression / Génération PDF
1. Cliquez sur l'icône **imprimante** sur un contrat
2. Un aperçu de la facture/contrat s'affiche dans un modal :
   - En-tête avec le logo et les informations de l'agence
   - Informations du locataire (nom, permis, téléphone)
   - Informations du véhicule (marque, modèle, matricule, compteurs)
   - Tableau détaillé : description, tarif, jours, total
   - Lignes d'extras et de pénalités si applicable
   - Récapitulatif : sous-total, remise, montant net dû, montant payé, reste dû
   - Informations de paiement (mode et statut)
   - Espaces pour les signatures (locataire et agence)
3. Cliquez sur **Générer PDF** pour télécharger le document au format PDF

### Modifier un contrat
- Cliquez sur l'icône **crayon** pour modifier un contrat
- En mode édition, le client et le véhicule sont affichés en lecture seule (non modifiables)
- Les autres champs peuvent être ajustés

### Supprimer un contrat
- Cliquez sur l'icône **poubelle** pour supprimer un contrat
- Une confirmation est demandée

## Module 5 — Maintenance

### Accès
- Route : `/maintenance`
- Menu de navigation : **Maintenance** (icône d'engrenage)

### Liste des interventions
Les interventions de maintenance sont affichées dans un **tableau** avec les colonnes :
- Véhicule (marque, modèle + matricule)
- Type d'intervention (Préventive, Corrective, Réparation accident, Inspection)
- Dates (date réalisée + prochaine date prévue)
- Kilométrage à la maintenance
- Coût total
- Atelier / garage
- Facture (lien vers le fichier PDF uploadé)
- Statut (Planifié, En cours, Terminé)
- Actions (éditer, supprimer)

### Enregistrer une intervention
1. Cliquez sur **+ Enregistrer une intervention**
2. Remplissez le formulaire :
   - **Véhicule** : Sélectionnez dans la liste déroulante
   - **Type d'intervention** : Préventive, Corrective, Réparation accident, Inspection
   - **Date de l'intervention** et **Prochaine maintenance prévue**
   - **Compteur km** à la maintenance
   - **Statut** : Planifié, En cours, Terminé
   - **Coûts** : Main-d'œuvre et Pièces (affichés côte à côte)
   - **Atelier** : Nom, Contact, Adresse
   - **Facture** : N° de facture + fichier PDF à uploader
   - **Description** des travaux effectués
3. Cliquez sur **Enregistrer**

### Comportement automatique
- Quand le statut d'une intervention passe à **En cours** → le statut du véhicule passe à **En maintenance**
- Quand l'intervention est **Terminée** → le statut du véhicule revient à **Disponible**

### Calendrier de maintenance
1. Cliquez sur le bouton **Calendrier** en haut de la page
2. Un modal s'ouvre avec une **timeline verticale** des événements de maintenance à venir :
   - Date de la prochaine maintenance
   - Titre de l'intervention
   - Matricule du véhicule
   - Badge du type d'intervention

## Module 6 — Consommables (Vidange & Filtres)

### Accès
- Route : `/consumables`
- Menu de navigation : **Consommables** (icône de clé à molette)

### Vue d'ensemble
Ce module permet le suivi dédié des remplacements récurrents de pièces d'usure. Tous les véhicules de la flotte sont listés avec l'état de chaque consommable.

### Types de consommables suivis
Les consommables par défaut (configurables dans les paramètres) :
1. **Vidange huile** (OilChange) — Intervalle : 10 000 km / 12 mois
2. **Filtre à huile** (OilFilter) — Intervalle : 10 000 km / 12 mois
3. **Filtre à air** (AirFilter) — Intervalle : 20 000 km / 24 mois
4. **Filtre à carburant** (FuelFilter) — Intervalle : 30 000 km / 24 mois
5. **Filtre habitacle** (CabinFilter) — Intervalle : 20 000 km / 12 mois
6. **Freins avant** (FrontBrakes) — Intervalle : 40 000 km
7. **Freins arrière** (RearBrakes) — Intervalle : 60 000 km
8. **Pneus avant** (FrontTires) — Intervalle : 50 000 km / 48 mois
9. **Pneus arrière** (RearTires) — Intervalle : 50 000 km / 48 mois
10. **Batterie** (Battery) — Intervalle : 36 mois

### Enregistrer un remplacement
1. Dans le module Consommables, sélectionnez un véhicule
2. Cliquez sur **+ Ajouter un remplacement**
3. Remplissez le formulaire :
   - Type de consommable
   - Date du remplacement
   - Compteur km au remplacement
   - Détails spécifiques (type d'huile/viscosité pour la vidange, marque/taille pour les pneus, capacité pour la batterie)
   - Notes
4. Cliquez sur **Enregistrer**

### Système d'alertes
- **OK** (vert) : Le consommable est dans les limites d'intervalle
- **Avertissement** (orange) : Le consommable approche de sa limite de remplacement
- **Dû** (rouge) : Le remplacement est en retard

Les seuils d'alerte sont configurables dans les Paramètres :
- Alerte km avant échéance : par défaut 1 000 km
- Alerte jours avant échéance : par défaut 30 jours

## Module 7 — Assurances & Contrôle Technique

### Accès
- Route : `/insurance-inspections`
- Menu de navigation : **Assurances & CT** (icône de bouclier)

Ce module est également accessible depuis le **Dossier du véhicule** (onglets Assurances et Contrôle Technique dans la modale de détails).

### Gestion des assurances
Chaque véhicule peut avoir une ou plusieurs polices d'assurance :
- **Assureur** : Nom de la compagnie d'assurance

- **Dates** : Date d'effet et date d'expiration
- **Prime** et **Valeur assurée**
- **Contact de l'agent**

### Statuts d'assurance

- **Bientôt expiré** (orange) : Expiration dans les 30/15/7 jours (seuils configurables)

### Gestion du contrôle technique
Chaque véhicule peut avoir un historique d'inspections techniques :
- **Date d'inspection** et **Date d'expiration**
- **Résultat** : Favorable, Contre-visite, Défavorable
- **Centre** : Nom et adresse du centre de contrôle
- **Coût** de l'inspection
- **Remarques** et **Document PDF**

### Statuts du contrôle technique
- **Valide** (vert) : Contrôle technique valide
- **Bientôt expiré** (orange) : Expiration prochaine
- **Expiré** (rouge) : Contrôle technique expiré
- **Échoué** (rouge) : Résultat défavorable

## Module 8 — Suivi du Carburant

### Accès
- Route : `/fuel`
- Menu de navigation : **Carburant** (icône de filtre/essence)
- Également accessible depuis le **Dossier du véhicule** (onglet Carburant)

### Journal des pleins
Chaque plein de carburant enregistre :
- **Date** du plein
- **Compteur km** (valeur de l'odomètre)
- **Volume** en litres
- **Coût par litre** et **Coût total** (calculé automatiquement)
- **Station** de service
- **Type de carburant**
- **Anomalie** : Indicateur si la consommation est anormale

### Calcul de la consommation
La consommation en **L/100km** est calculée automatiquement entre deux pleins successifs :
- Consommation = (Litres du plein actuel) / (Km actuel − Km précédent) × 100

### Détection d'anomalies
Le système compare la consommation de chaque plein à la moyenne du véhicule. Si la consommation dévie significativement (plus de 50% au-dessus de la moyenne), un **badge rouge "Surconsommation"** est affiché.

### Export
Le journal de carburant peut être exporté en **CSV** via le bouton dédié.

## Module 9 — Suivi du Kilométrage

### Fonctionnement
Le suivi kilométrique est transversal à l'application. Les entrées kilométriques sont créées :
- **Automatiquement** : Au départ d'un contrat de location (source : "Départ contrat")
- **Automatiquement** : Au retour d'un contrat de location (source : "Retour contrat")
- **Manuellement** : Par l'administrateur via le dossier du véhicule

### Accès
- Depuis le **Dossier du véhicule** → onglet **Historique Kilométrage**
- La timeline affiche toutes les entrées kilométriques triées par date
- Chaque entrée est codée par couleur selon la source

### Ajout manuel
1. Dans le dossier du véhicule, onglet Historique Kilométrage
2. Cliquez sur **+ Relevé**
3. Saisissez : Date, Compteur km, Notes
4. Cliquez sur **Valider**

### Export
L'historique kilométrique peut être exporté en **CSV**.

### Alerte d'inactivité
Si un véhicule n'a eu aucune activité (location ou entrée manuelle) pendant plus de N jours (configurable, par défaut 15 jours), une alerte d'inactivité est générée dans le Centre d'alertes.

## Module 10 — Centre d'Alertes

### Accès
- Route : `/alerts`
- Menu de navigation : **Alertes** (icône de triangle d'avertissement)

### Vue d'ensemble
Le Centre d'alertes centralise toutes les alertes actives de tous les modules.

### KPI en haut de page
Quatre cartes cliquables pour filtrer par sévérité :
- **Total** : Nombre total d'alertes
- **Critique** (rouge avec animation pulsante) : Alertes nécessitant une action immédiate
- **Avertissement** (orange) : Alertes approchant d'une échéance
- **Information** (bleu) : Alertes informatives

### Filtres
- **Recherche** par texte libre
- **Module** : Tous, Assurance, Contrôle technique, Maintenance, Inactivité km, Permis de conduire, Consommable

### Tableau des alertes
Colonnes :
- **Sévérité** : Badge Critique / Avertissement / Info
- **Type** : Badge coloré avec icône (Assurance, Contrôle technique, Inactivité km, Permis, Consommable, Maintenance)
- **Concerne** : Lien cliquable vers le véhicule ou le client concerné
- **Description** : Message détaillé de l'alerte
- **Échéance** : Nombre de jours restants ou de retard
- **Date** : Date de l'événement concerné

### Types d'alertes générées
1. **Assurance** : Police d'assurance expirée ou bientôt expirée
2. **Contrôle technique** : Inspection technique expirée ou bientôt expirée
3. **Maintenance** : Maintenance planifiée arrivant à échéance
4. **Consommable** : Remplacement de consommable dû ou approchant
5. **Inactivité kilométrique** : Véhicule sans activité depuis N jours
6. **Permis de conduire** : Permis d'un client expiré

### Pagination
Les alertes sont paginées si leur nombre dépasse la limite par page.

## Module 11 — Rapports & Analyses

### Accès
- Route : `/reports`
- Menu de navigation : **Rapports** (icône de graphique)

### Indicateurs clés (KPI)
Trois cartes en haut de page :
1. **Chiffre d'affaires brut** : Revenu total, avec décomposition payé (vert) vs impayé (rouge)
2. **Taux d'utilisation moyen** : Pourcentage moyen de temps pendant lequel les véhicules sont loués
3. **Taille de la flotte** : Nombre total de véhicules, avec les sous-totaux "loués" et "en garage"

### Graphiques
1. **Disponibilité de la flotte** : Graphique en anneau (doughnut) montrant la répartition des statuts des véhicules
2. **Recouvrement financier** : Graphique en anneau montrant la proportion payé vs impayé
   - Filtrable par plage de dates (date de début et date de fin)

### Analyse de rentabilité
Tableau détaillé de la rentabilité **par véhicule** :
- **Véhicule** : Marque et modèle
- **Matricule** : Numéro de plaque
- **Taux d'utilisation** : Pourcentage de jours loués sur la période

- Nombre de locations
- Chiffre d'affaires total généré

### Paiements en retard

- Client
- Véhicule

- Statut de paiement (badge Impayé / Partiellement payé)
- Paginé (5 lignes par page)

## Flux de travail typiques

### Flux 1 : Louer un véhicule à un client
1. **Vérifier la disponibilité** : Aller dans Véhicules, filtrer par statut "Disponible"
2. **Vérifier ou créer le client** : Aller dans Clients, chercher le client ou en créer un nouveau
3. **Créer le contrat** : Aller dans Contrats → + Nouveau contrat
4. Sélectionner le client et le véhicule
5. Définir les dates, le tarif, le mode de paiement
6. Choisir le statut : "Activer maintenant" pour activer la location immédiatement
7. Le véhicule passe automatiquement au statut "Loué"

### Flux 2 : Retourner un véhicule
1. Aller dans Contrats
2. Trouver le contrat actif (filtre "Actif")
3. Cliquer sur le bouton vert **Retour**
4. Saisir le compteur km au retour et la date/heure réelle
5. Ajouter les pénalités éventuelles (carburant, dommages)
6. Cocher "Envoyer en maintenance" si nécessaire
7. Valider le retour
8. Le véhicule revient à "Disponible" (ou "En maintenance")
9. Imprimer la facture si besoin

### Flux 3 : Planifier et suivre la maintenance
1. Aller dans Maintenance → + Enregistrer une intervention
2. Sélectionner le véhicule, le type d'intervention
3. Définir la date et la prochaine date prévue
4. Quand le statut passe à "En cours", le véhicule est automatiquement marqué "En maintenance"
5. Une fois terminée, le véhicule revient à "Disponible"
6. Consulter le Calendrier pour voir les interventions à venir

### Flux 4 : Surveiller les alertes
1. Consulter régulièrement le Centre d'alertes
2. Filtrer par sévérité ou par module pour prioriser
3. Agir sur les alertes critiques en priorité :
   - Renouveler les assurances expirées
   - Planifier les contrôles techniques
   - Effectuer les remplacements de consommables
   - Contacter les clients dont le permis est expiré

### Flux 5 : Analyser la rentabilité
1. Aller dans Rapports
2. Consulter les KPI financiers en haut de page
3. Examiner l'analyse de rentabilité par véhicule
4. Identifier les véhicules avec un bénéfice net négatif (coûts supérieurs aux revenus)
5. Exporter le rapport en CSV pour une analyse approfondie
6. Consulter les top clients et les paiements en retard

## API REST — Points d'entrée principaux

### Authentification
- `POST /api/auth/login` — Connexion (retourne un JWT)
- `POST /api/auth/register` — Créer un nouvel administrateur
- `PUT /api/auth/profile` — Mettre à jour le profil
- `PUT /api/auth/change-password` — Changer le mot de passe

### Véhicules
- `GET /api/vehicle` — Liste paginée avec filtres (type, status, fuelType, search, page, pageSize)
- `GET /api/vehicle/{id}` — Détail d'un véhicule
- `POST /api/vehicle` — Créer un véhicule
- `PUT /api/vehicle/{id}` — Modifier un véhicule
- `DELETE /api/vehicle/{id}` — Supprimer un véhicule
- `POST /api/vehicle/upload-photo` — Uploader une photo

### Clients
- `GET /api/client` — Liste avec recherche
- `GET /api/client/{id}` — Détail d'un client
- `POST /api/client` — Créer un client
- `PUT /api/client/{id}` — Modifier un client
- `DELETE /api/client/{id}` — Supprimer un client

### Contrats
- `GET /api/contract` — Liste paginée avec filtres (status, paymentStatus, search, page, pageSize)
- `GET /api/contract/{id}` — Détail d'un contrat
- `POST /api/contract` — Créer un contrat
- `PUT /api/contract/{id}` — Modifier un contrat
- `PUT /api/contract/{id}/return` — Retourner un véhicule (clôturer le contrat)
- `DELETE /api/contract/{id}` — Supprimer un contrat

### Kilométrage
- `GET /api/km/vehicle/{vehicleId}` — Historique km d'un véhicule
- `POST /api/km` — Ajouter une entrée km manuelle

### Maintenance
- `GET /api/maintenance` — Liste de toutes les interventions
- `GET /api/maintenance/calendar` — Événements du calendrier
- `POST /api/maintenance` — Créer une intervention
- `PUT /api/maintenance/{id}` — Modifier une intervention
- `DELETE /api/maintenance/{id}` — Supprimer une intervention
- `POST /api/maintenance/upload-invoice` — Uploader une facture

### Consommables
- `GET /api/consumable/vehicle/{vehicleId}/status` — État des consommables d'un véhicule
- `GET /api/consumable/config` — Configuration des intervalles
- `PUT /api/consumable/config` — Modifier la configuration
- `POST /api/consumable/log` — Ajouter un remplacement

### Assurances
- `GET /api/insurance/vehicle/{vehicleId}` — Polices d'un véhicule (avec statut)
- `POST /api/insurance` — Créer une police
- `POST /api/insurance/upload-doc` — Uploader un document

### Contrôle technique
- `GET /api/technicalinspection/vehicle/{vehicleId}` — Inspections d'un véhicule (avec statut)
- `POST /api/technicalinspection` — Créer une inspection
- `POST /api/technicalinspection/upload-doc` — Uploader un document

### Carburant
- `GET /api/fuel/vehicle/{vehicleId}` — Journal carburant d'un véhicule (avec consommation calculée)
- `POST /api/fuel` — Ajouter un plein

### Alertes
- `GET /api/alert` — Toutes les alertes actives

### Rapports
- `GET /api/report/fleet-status` — Statuts de la flotte
- `GET /api/report/revenue` — Revenus (avec filtres de dates)
- `GET /api/report/utilization` — Taux d'utilisation par véhicule
- `GET /api/report/profitability` — Rentabilité par véhicule
- `GET /api/report/top-clients` — Top clients
- `GET /api/report/unpaid-contracts` — Contrats impayés

### Paramètres
- `GET /api/settings` — Récupérer les paramètres globaux
- `PUT /api/settings` — Mettre à jour les paramètres

## Glossaire

| Terme | Définition |
|---|---|
| **Matricule** | Numéro de plaque d'immatriculation du véhicule |
| **VIN** | Vehicle Identification Number — numéro de châssis unique |
| **Kilométrage** | Distance parcourue en kilomètres, mesurée par le compteur (odomètre) |
| **Compteur km** | Valeur actuelle de l'odomètre du véhicule |
| **Tarif journalier** | Prix de location par jour |
| **Caution** | Dépôt de garantie versé par le locataire |
| **Vidange** | Changement d'huile moteur |
| **Contrôle technique** | Inspection réglementaire périodique du véhicule |
| **TCO** | Total Cost of Ownership — coût total de possession d'un véhicule |
| **L/100km** | Consommation de carburant en litres pour 100 kilomètres |
| **Brouillon** | Contrat créé mais pas encore activé (réservation) |
| **JWT** | JSON Web Token — jeton d'authentification |
